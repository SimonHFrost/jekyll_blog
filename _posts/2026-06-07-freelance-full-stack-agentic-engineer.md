I build software end to end — and increasingly, so do the agents I work alongside. This is a note on what "full stack" means when a good chunk of the stack is now written, reviewed, and orchestrated with AI.

## What "agentic" actually means here

Not "I use an autocomplete." Agentic engineering is handing a goal to a system that can plan, take actions, check its own work, and iterate — and knowing where the human still has to sit in that loop.

- Planning and decomposition before code
- Tools, not just text: agents that read files, run tests, hit APIs
- Verification loops — the agent proves the change works
- Knowing when to take the wheel back

## The freelance angle

Why this matters for clients:

- Faster prototypes, shorter feedback loops
- Smaller, reviewable changes — git commits as save points
- Quality tooling (tests, linting) does double duty as guardrails for AI
- You still get a human who'll put their name behind every line
